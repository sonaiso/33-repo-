## One Auditable Claim

Slot Geometry is not a table of static slots. It is a layered contract system
for GPT-answer reasonableness verification.

```text
SlotGeometryLayer =
  ConditionOfPossibility
  + MinimumCompleteLimit
  + Opening
  + Demand
  + IdentityPreservation
  + Closure
  + Residual
  + LicensedTransition
```

Arabic statement:

```text
الطبقة =
  شرط إمكان
  + حد أدنى مكتمل
  + فتح
  + طلب
  + حفظ هوية
  + إغلاق
  + بقية
  + انتقال مرخّص
```

If a unit cannot answer these eight questions, it remains raw material,
possibility, or candidate. It is not a licensed layer.

## Scope

This document establishes the foundational audit law for treating every
linguistic unit as a contract-bound gate rather than as an isolated symbol.

It binds future Slot Geometry work to the project objective:

```text
GPT answer
  → L0 structural material
  → L1 formal description
  → L2 comparison and reasoning, only after explicit opening
  → L3 reasonableness verdict, only after explicit opening
```

Linguistic analysis is therefore a condition of possibility for checking GPT
answer reasonableness. It is not an independent final objective.

## The Eight Contract Questions

Every proposed Slot Geometry gate must answer all eight questions before it can
be treated as a licensed layer.

| Contract field | Arabic field | Required audit answer |
|---|---|---|
| `ConditionOfPossibility` | شرط الإمكان | What must exist before this unit is possible? |
| `MinimumCompleteLimit` | الحد الأدنى المكتمل | What is the smallest complete form of the unit? |
| `Opening` | الفتح | What does this unit open after completion? |
| `Demand` | الطلب | What does this unit require to complete its relation? |
| `IdentityPreservation` | حفظ الهوية | What identity must remain preserved across the gate? |
| `Closure` | الإغلاق | When is this gate closed enough to stop demanding? |
| `Residual` | البقية | What remains visible if the gate is incomplete or blocked? |
| `LicensedTransition` | الانتقال المرخّص | What exactly permits movement to the next gate? |

No answer may be replaced by silence, probability, context alone, or a bare
assertion. Residuals remain visible until discharged by a later licensed gate.

## Residual Policy

Residuals are not failures to be hidden. They are required audit outputs when a
gate is incomplete, blocked, ambiguous, or not yet licensed.

Examples of residual vocabulary:

- `LETTER_WITHOUT_LICENSED_VOWEL`
- `WAQF_WASL_UNRESOLVED`
- `SYLLABLE_INCOMPLETE`
- `ROOT_AUGMENT_DISTINCTION_UNRESOLVED`
- `TOOL_WITHOUT_REQUIRED_OPERAND`
- `REFERENCE_UNRESOLVED`
- `GOVERNOR_WITHOUT_GOVERNED`
- `RELATION_NOT_CLOSED`
- `IFADAH_NOT_LICENSED`
- `HUKM_NOT_LICENSED`
- `TRUTH_NOT_PROVEN`

These names are audit vocabulary only. They do not replace `FailureCode`.

## NeedGate Boundary

No morphology, syntax, semantics, relation analysis, ifādah analysis, hukm
analysis, or truth analysis opens merely because the next item exists in the
chain.

Each opening requires a verification need:

```text
NeedGate =
  specific GPT claim under review
  + named linguistic or reasoning need
  + licensed current gate
  + visible residual if the need cannot be discharged
```

Analysis is demand-driven by the verification need, not exhaustive.
