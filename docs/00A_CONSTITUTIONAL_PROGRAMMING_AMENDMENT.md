# Constitutional Programming Amendment — `docs/00A_CONSTITUTIONAL_PROGRAMMING_AMENDMENT.md`

## Authority
Derived from `docs/00_MAQOOL_CONSTITUTION.md §5` and `docs/02_L1_META_LANGUAGE_BOUNDARY.md`.

## Scope
This amendment governs executable constitutional programming inside **L1 only**.
It does not open L2 or L3.

## Constitutional Programming Axiom

No executable entity may enter the system unless it is domain-scoped,
origin-bound, proof-carrying, trace-preserving, and rank-capped.

No candidate without domain.
No concept without contract.
No contract without domain.
No domain without origin.
No origin without MRKProof.
No branch without BranchLicenseProof.
No operation without OperationSpec.
No gate without executable predicate.
No bridge without identity-preserving translator.
No decision without ProofObject.
No agent output without Kernel verdict.

## Explicit Prohibitions

No Boolean as proof.
No manual verdicts.
No rank promotion.
No textual gate as gate.
No textual bridge as bridge.
No evidence list as proof.
No identity_preserved=True by default.
No coverage dashboards with manual totals.

## L1 Output Envelope

All L1 constitutional programming outputs must remain `rank = "CANDIDATE"`.
L1 outputs are bounded to:
- Candidate
- BlockedCandidate
- ResidualCandidate
- ProofCandidate

Forbidden in this phase:
- FinalMeaning
- FinalSyntaxRole
- Ifadah
- Hukm
- Tanzil
