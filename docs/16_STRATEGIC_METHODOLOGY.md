## §1. وصف المشروع

### الاسم
**تعقُّل (Taaqol-GPT)** — محرك تحليل لغوي عربي محكوم دستوريًا

### الرسالة
بناء نظام يُحلّل اللغة العربية عبر أربع طبقات متسلسلة (L0→L1→L2→L3) بحيث:
- كل كيان مُجمّد (immutable) ومُتتبّع دستوريًا
- كل رفض مُسمّى بكود فشل واضح
- كل دالة نقيّة (pure) بلا آثار جانبية
- كل انتقال بين الطبقات مُرخّص ومحفوظ الهوية

### الفلسفة
> **"لا كود بلا أصل، ولا فرع بلا ترخيص، ولا انتقال بلا إثبات"**

## §8. الخطة التنفيذية

### المرحلة الحالية: إغلاق L0 + فتح L1

```
[الآن] ─── ST1..ST6 ───► [PR-9: إغلاق L0] ───► [PR-10..13: L1] ───► [PR-14: إغلاق L1]
                              │                        │
                         KPI-01..07 يمر              KPI-P2 يتقدم
```

### آلية القياس

1. **قبل كل PR**: تشغيل `pytest tests/test_kpi_indicators.py` — يجب أن تمر جميع KPIs
2. **بعد كل PR**: تحديث جدول §7 بالقيم الفعلية
3. **عند الإغلاق**: تشغيل CI guard الكامل + اختبارات المرحلة

### معادلة التقدم

```
التقدم% = (عدد KPIs المُحققة / إجمالي KPIs) × 100

التقدم الحالي:
  هيكلي: 7/7 = 100% ✅
  تقدمي: 1/5 = 20% (L0 فقط مكتمل)
  إجمالي: 8/12 = 67%
```

## §10. قواعد الحوكمة المنهجية

| القاعدة | الشرح | الإنفاذ |
|---------|-------|---------|
| لا كود بلا هدف | كل سطر يُرجع لهدف في §3-§6 | CI guard + trace_ref |
| لا هدف بلا مؤشر | كل هدف له KPI قابل للقياس | جدول §7 |
| لا مؤشر بلا اختبار | كل KPI له test function | `test_kpi_indicators.py` |
| لا اختبار بلا نتيجة | كل اختبار ينتهي بـ pass/fail | pytest |
| لا تقدم بلا إثبات | التقدم = عدد KPIs المُحققة فقط | معادلة §8 |
