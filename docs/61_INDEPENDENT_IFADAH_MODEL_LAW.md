## Scope

The independent ifādah model describes how a closed or closeable utterance
relation may be classified as minimally informative in its maqam.

It is limited to this audit question:

```text
Did the utterance perform a minimally complete speech function in its context?
```

## Non-scope

This model does not:

- prove truth;
- issue hukm;
- issue tanzil;
- create yaqīn;
- certify reality;
- create a decision engine;
- create runtime predicates;
- create runtime translators;
- open `D5_IFADAH` as a runtime domain;
- open `L2` or `L3`.

## Preserved Prerequisites

The model assumes that prior constitutional gates remain prior and preserved:

- taʿyīn: the utterance carrier, locus, boundary, and reference are identified;
- tamyīz: the utterance type and focal function are distinguished;
- sabab: a speech-function cause opens intelligibility;
- sharṭ: required parts for that type are present or licensed by maqam;
- māniʿ: no blocking absence prevents qualified understanding;
- waṣf muʾaththir: the effective feature closes a functional gap;
- farq qādiḥ: no defeating difference cuts the proposed function.

These are not re-founded here. They remain prior conditions and are used only
as audit vocabulary for `D5_IFADAH`.

## Ifādah Definition

Ifādah is the minimal completion of a speech function in its maqam so that a
qualified addressee knows what the utterance is doing.

```text
Ifādah is established only when:
  Required positives:
    - utterance candidate
    - utterance type
    - focal function
    - required parts
    - lexical or maqam-based closure
    - visible classified residuals
  Required negatives:
    - no ifādah blocker
```

Ifādah is not truth. Ifādah is not hukm. Ifādah is not yaqīn. Ifādah is not the
mere adjacency of words.

## Eight Initial Utterance Types

| Type | Arabic | Focal function |
|------|--------|----------------|
| report | خبر | predicative relation assigned to a locus |
| request | طلب | requested act from an addressee or intended direction |
| question | سؤال | unknown answer sought |
| address | نداء | alerting or summoning the addressed one |
| condition | شرط | suspending an answer on a condition |
| answer | جواب | satisfying a prior question or question-like maqam |
| supplication | دعاء | request directed to Allah |
| restraint | زجر | urgent prevention, rebuke, or stopping |

These eight are initial operating doors only. They are not an exhaustive
runtime grammar of Arabic.

## Minimal Ifādah Conditions by Type

| Type | Minimal condition | Ifādah blocker |
|------|-------------------|----------------|
| report | locus + predicate + intelligible relation | missing relation or blocking reference |
| request | requested act + licensed addressee | unknown requested act or addressee |
| question | question form + questioned matter + expected answer shape | unknown object of inquiry |
| address | address marker or maqam + explicit or licensed addressee | missing addressee without maqam license |
| condition | condition marker or sense + condition content + answer or sufficient maqam closure | condition without answer |
| answer | prior question or demanding maqam + answer that satisfies the demand | answer misses the question focus |
| supplication | requested matter + direction to Allah + beneficiary or target | disturbed request or direction |
| restraint | restraint form or maqam + restrained party or act + stopping effect | missing restrained act without context |

## Cause, Condition, Blocker

The cause of ifādah is the presence of a focal speech function.

The condition of ifādah is completion of the required parts of the utterance
type, either explicitly or through licensed ellipsis.

The blocker of ifādah is any required absence that prevents a qualified
addressee from understanding the utterance function in its maqam.

Common blockers include:

1. unknown utterance type;
2. unfixed focal function;
3. missing required part;
4. unlicensed ellipsis;
5. unresolved required reference;
6. open operator requiring completion;
7. maqam unable to close the gap;
8. answer that does not satisfy the question;
9. defeating difference against the proposed function.

## Ellipsis License

Ellipsis may be supplied only by a sufficient lexical or maqam-based license:

```text
Licensed ellipsis requires a prior expression, question, maqam, usage custom,
or structural necessity that makes the supplied completion fixed or dominant
enough for minimal ifādah.
```

Mental possibility alone is not a license.

## Maqam Boundary

The maqam does not create ifādah from nothing. It closes or prefers what already
has a carrier:

- expression;
- prior question;
- visible situation;
- usage custom;
- apparent relation;
- preserved trace.

```text
Maqam is a cause for understanding, not truth.
Maqam is a condition for closure, not hukm.
```

## Four Ifādah Audit Outcomes

| Outcome | Meaning |
|---------|---------|
| `COMPLETE_IFADAH` | the utterance completes its function by wording alone |
| `MAQAMI_COMPLETE_IFADAH` | maqam closes a visible functional gap |
| `SUSPENDED_IFADAH` | type and focus appear, but a required part remains open |
| `NO_IFADAH` | no understandable speech function is reached |

These labels are audit outcomes only. They are not computed verdicts and do
not authorize a runtime evaluator.

## Acceptance and Rejection Examples

### Condition without answer

```text
إن جاء زيد...
```

Audit result: `SUSPENDED_IFADAH`, because condition is opened but the answer is
not present and no maqam closure is supplied.

### Address without addressee

```text
يا...
```

Audit result: `SUSPENDED_IFADAH` or `NO_IFADAH`, depending on whether maqam
licenses the missing addressee.

### Unknown reference

```text
هو فعل ذلك.
```

Audit result: `SUSPENDED_IFADAH` when the required reference is unresolved and
could be closed by maqam; otherwise no complete ifādah is available.

### Answer that misses the question

```text
Q: من حضر؟
A: الكتاب جديد.
```

The answer may be a complete report by itself, but it fails as an answer
because it does not satisfy the question focus.

## Final Boundary Law

No ifādah is established until the utterance type is fixed, its focal function
is identified, its required parts are complete by expression or licensed
ellipsis, blocking reference is absent, maqam closes only what it is licensed to
close, and no required operator remains open.

If the speech function completes, ifādah is present. If a required functional
gap remains, ifādah is suspended or absent. If the gap concerns truth, evidence,
or certainty, then ifādah may be present while hukm remains deferred.
